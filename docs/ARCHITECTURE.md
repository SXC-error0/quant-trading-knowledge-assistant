# 系统架构

## 1. 目标

本服务作为看盘工具的独立 AI 知识问答后端，优先解决“基于专业资料回答问题并给出来源”的能力，不承接自动交易执行。

## 2. 数据流

```text
外部切片系统输出 JSONL
        ↓
POST /api/v1/knowledge/import
        ↓
Embedding API 生成向量
        ↓
Qdrant 保存向量与来源元数据
        ↓
用户问题 POST /api/v1/chat/ask
        ↓
风险边界识别
        ├── 触发越界 → 返回受约束解释
        └── 普通知识问题 → Embedding 查询
                              ↓
                       向量召回 + 轻量词项加权
                              ↓
                       LLM 基于知识片段回答
                              ↓
                       返回答案与来源
```

## 3. 知识域

| Domain | 用途 |
| --- | --- |
| `trading_books` | 交易理论、指标、策略、回测与风控知识 |
| `product_manual` | 产品页面、操作流程与 FAQ |
| `signal_rules` | 提醒类型、触发解释和风险含义 |

## 4. 当前检索链路

MVP 使用向量召回并对与问题重合的关键词进行轻量加权。这是便于快速接通数据和验证问答效果的基线实现。

正式版本建议升级为：

```text
意图路由 → 向量召回 + BM25 全文召回 → Reranker → 生成回答
```

## 5. 与看盘前端的集成

前端仅需调用 `/api/v1/chat/ask`。后续可增加会话 ID、用户反馈、引用展开和信号详情工具接口。模型不会接收或保管用户交易所 Secret。
